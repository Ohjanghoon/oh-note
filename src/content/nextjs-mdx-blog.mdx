### 📝 개발 스택 선정
위에 기획한 기능들을 구현하기 위해 가장 우선시한 기준은 **"개발이 편리한가?"**, **"빠른 개발과 유지보수가 가능한가?"** 였습니다.

이러한 기준을 바탕으로 아래와 같은 특징들을 고려하여 개발 스택을 선정하였습니다.

<br/>
###### 1. Framework: [Next.js 15.2](https://nextjs.org/)(with. TypeScript)

제 포트폴리오 사이트는 **React 기반**으로 제작했지만, 블로그의 경우 **SEO 최적화, 빠른 페이지 로딩, 서버 기능 지원**이 필요해 Next.js를 선택하게 되었습니다. 가장 최신 버전을 접해보고 13버전 이후부터 새로 도입된 App Router를 학습하고자 Next.js 15버전을 사용하였습니다.


1. **SEO 최적화**
    - 블로그는 검색 엔진 유입이 중요하므로, 검색 노출도를 높이기 위해 <b>Next.js의 SSG(Static Site Generation)</b>를 활용하기로 하였습니다.
2. **빠른 로딩 속도**
    - SSG를 통해 정적 페이지를 미리 생성하여 **React CSR보다 성능이 우수**.
3. **서버 기능 활용**
    - API Routes를 이용해 **조회수, 좋아요, 방명록 기능**구현 필요.
4. **배포 및 유지보수 용이**
    - **Vercel과의 최적화된 호환성** 덕분에 배포 및 관리가 간편합니다.

<Callout type="tip">
SSG를 활용하면 검색 노출도가 높아지는 이유가 뭘까요?
<br/>
SSG를 사용하면 페이지가 요청될 때마다 새로 생성되는 것이 아닌, **빌드 시 미리 생성된 정적 페이지를 빠르게 제공**하여 검색 엔진이 쉽게 읽을 수 있습니다.
또한, 빠른 로딩 속도로 인해 SEO 점수가 향상되어 검색 노출도를 올릴 수 있습니다.
<br/>
(+) 검색 엔진은 정적인 HTML 페이지를 선호하며, JavaScript가 필요 없는 페이지일수록 색인(Indexing) 속도가 빠릅니다.
</Callout>
<br/>
<br/>
###### 2. Blog Posting: [MDX](https://mdxjs.com/)

간편한 글 작성과 유연한 확장성을 고려하여 MDX 파일 기반으로 블로그 게시글을 포스팅하기로 하였습니다.

1. Markdown의 간단한 문법으로 글 작성이 가능하고, 컴포넌트를 직접 삽입하여 코드블록과 같은 인터랙티브한 요소를 추가할 수 있음.
2. 파일 기반으로 관리하여 유지보수가 용이.
3. 필요에 따라 다크모드와 코드 하이라이팅 등 다양한 기능으로 확장 가능.

<br/>
<br/>
###### 3. Style: [Tailwind CSS 4.0](https://tailwindcss.com/)

블로그의 스타일링은 다음과 같은 장점으로 Tailwind CSS를 활용하였습니다.

1. Tailwind CSS의 직관적인 클래스 네이밍과 유지보수의 편의성
2. UI의 일관성 유지
3. 브레이크 포인트에 따른 반응형 디자인 적용 가능
4. 다크 모드 지원

<br/>
<br/>
###### 4. Database: [Supabase](https://supabase.com/)

PostgreSQL 기반으로 한 Supabase는 이번 블로그 개발을 하며 처음 접해본 서비스인데요.

1. 간단한 설정과 직관적인 조작법
2. 서버리스 데이터베이스 지원이 가능

한 점이 가장 마음에 들었습니다. 데이터베이스가 필요한 **익명 방명록과 게시글 좋아요 기능**을 구현할 때 유용하다고 판단되어 선택하게 되었습니다.

<br/>
<br/>
###### 5. Deployment: [Vercel](https://vercel.com/)

1. Next.js 프로젝트에 최적화된 호환성
    - Next.js를 만든 Vercel에서 제공하는 배포 플랫폼으로, 별도의 설정 없이도 **자동 최적화된 빌드 및 캐싱**이 적용
2. CI/CD 및 GitHub Branch Tracking
    - GitHub와 연동하여 브랜치 푸쉬만으로 자동 배포가 이루어짐.
    - 브랜치 별로 미리보기(Preview Deployment) 환경을 제공하여 변경사항을 확인하고 서비스를 배포할 수 있음.
3. 내장된 Analytics 기능
    - 복잡한 설정 없이 Vercel Analytics를 통해 실시간 트래픽 및 페이지 뷰 수를 모니터링 할 수 있음.
